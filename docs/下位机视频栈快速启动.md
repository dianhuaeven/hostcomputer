# 下位机视频栈快速启动

本文用于在下位机上快速启动视频推流和上位机 bridge。当前正式入口只有一个：

```bash
/home/rera/hostcomputer/ros1_bridge/video_stack.sh
```

需要理解生命周期和单路控制时看《下位机视频栈使用文档》；需要改摄像头、RTSP 地址或 systemd 环境时看《下位机视频栈配置文档》。

## 1. 拉取最新代码

```bash
cd /home/rera/hostcomputer
git checkout main
git pull
```

确认入口脚本存在：

```bash
ls -l ros1_bridge/video_stack.sh
```

## 2. 检查配置

编辑视频源配置：

```bash
nano /home/rera/hostcomputer/ros1_bridge/video_sources.local.yaml
```

重点确认 `rtsp.public_host` 是下位机 NUC 在局域网内的 IP：

```yaml
rtsp:
  public_host: 192.168.1.83
  publish_host: 127.0.0.1
  port: 8554
  server: mediamtx
  transport: tcp
```

`public_host` 是上位机拉 RTSP 的地址；`publish_host` 是 FFmpeg 发布到本机 MediaMTX 的地址，通常保持 `127.0.0.1`。

## 3. 检查前置服务

确认 MediaMTX 正常：

```bash
systemctl status mediamtx --no-pager
ss -lntp | grep 8554
```

如果没有启动：

```bash
sudo systemctl start mediamtx
```

确认当前启用的 5 路摄像头设备名存在：

```bash
ls -l /dev/paw_realsense /dev/overheadcam /dev/pawcam /dev/frontcam /dev/armcam
```

`/dev/backcam` 当前在配置里保留但 `enabled: false`，缺失不会影响 5 路启动：

```bash
ls -l /dev/backcam 2>/dev/null || echo "/dev/backcam not present, ok while disabled"
```

确认 ROS master 已启动：

```bash
ss -lntp | grep 11311
```

如果没有启动，先在一个终端运行：

```bash
roscore
```

## 4. 启动视频栈

```bash
cd /home/rera/hostcomputer/ros1_bridge
ROS_SETUP=/home/rera/robot24_ws/devel/setup.bash ./video_stack.sh start
```

`ROS_SETUP` 用来加载 ROS Noetic 和工作空间的 Python/ROS 环境；如果当前终端已经 source 过同一个 setup，可以省略。

成功时应看到：

```text
MediaMTX: listening on 127.0.0.1:8554
video_manager_node: listening on 127.0.0.1:18081
host_bridge_node: listening on 127.0.0.1:9090
host_bridge_debug: listening on 127.0.0.1:18080
summary: online=5/5 desired=5 errors=0
```

## 5. 查看状态

```bash
cd /home/rera/hostcomputer/ros1_bridge
./video_stack.sh status
```

每路摄像头应类似：

```text
camera 0 paw_realsense online=True desired=True pid=9863 log=/tmp/rtsp_logs/paw_realsense.log error=
```

字段含义：

- `desired=True`：配置或请求要求这路保持在线。
- `online=True`：对应 FFmpeg publisher 当前正在运行。
- `pid`：这路 FFmpeg 子进程 PID。
- `log`：这路 FFmpeg 日志文件。
- `error`：最近一次错误。

## 6. 验证 RTSP

```bash
for p in paw_realsense overheadcam pawcam frontcam armcam; do
  echo "===== $p ====="
  ffprobe -v error -rtsp_transport tcp \
    -show_entries stream=codec_name,width,height,r_frame_rate \
    -of default=noprint_wrappers=1 \
    "rtsp://127.0.0.1:8554/$p"
done
```

正常应看到：

```text
codec_name=h264
width=960
height=720
r_frame_rate=30/1
```

## 7. 上位机连接信息

上位机连接 bridge：

```text
192.168.1.83:9090
```

上位机收到的 RTSP 地址应为：

```text
rtsp://192.168.1.83:8554/paw_realsense
rtsp://192.168.1.83:8554/overheadcam
rtsp://192.168.1.83:8554/pawcam
rtsp://192.168.1.83:8554/frontcam
rtsp://192.168.1.83:8554/armcam
```

## 8. 停止或重启

停止：

```bash
cd /home/rera/hostcomputer/ros1_bridge
./video_stack.sh stop
```

重启：

```bash
cd /home/rera/hostcomputer/ros1_bridge
ROS_SETUP=/home/rera/robot24_ws/devel/setup.bash ./video_stack.sh restart
```

## 9. 快速排障

如果 `errors` 不为 0：

```bash
./video_stack.sh status
tail -100 /tmp/rtsp_logs/<camera>.log
```

如果 `video manager unavailable`：

```bash
tail -200 /tmp/rtsp_logs/video_manager_node.log
python3 -c "import yaml; yaml.safe_load(open('/home/rera/hostcomputer/ros1_bridge/video_sources.local.yaml')); print('yaml ok')"
```

如果 bridge 没监听 `9090`：

```bash
tail -200 /tmp/rtsp_logs/host_bridge_node.log
ss -lntp | grep 11311
```

如果 MediaMTX 异常：

```bash
journalctl -u mediamtx --since '10 minutes ago' --no-pager
```
